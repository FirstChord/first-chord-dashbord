# Operations

Start here when running, deploying, diagnosing, or recovering the system.

- [Operations runbook](./runbook.md) — environments, credentials, deploys,
  integration health, backups, smoke checks, and rollback.
- [Disaster recovery](./disaster-recovery.md) — recovery from loss of Railway,
  Sheets, or the operating Mac.
- [Continuity](./continuity.md) — the people-side fortnight plan if Finn is
  unavailable.
- [Incident notes](./incidents/bug-fixes.md) — proven fixes and recovery clues.
- [WhatsApp bridge](./integrations/whatsapp-incoming-bridge.md) and
  [WordPress redirects](./integrations/wordpress-redirects.md) — focused

For a safety-sensitive integration, pair the runbook with its architecture,
policy, and workflow document rather than relying on one troubleshooting note.
